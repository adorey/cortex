# 🚀 Cortex v0.2.0 — Layered overrides

> *"I'd far rather be happy than right any day."* — Slartibartfast
>
> But sometimes you're both. Cortex now lets host projects extend any agent layer without forking the framework.

## TL;DR

A host project can now create **overlays** for `roles/`, `capabilities/`, `personalities/`, and `workflows/` — at workspace level and/or service level. The cascade pattern that has always existed for workflows is now generalized to all four layers.

## What's new

### 🏗️ Layered overrides (cascade for all agent layers)

Anchored by [ADR-001](docs/adr/ADR-001-layered-overrides.md). Read [docs/extending-layers.md](docs/extending-layers.md) for the practical guide.

**Resolution order** (most specific wins):

```
{service}/agents/{layer}/...                    ← priority 1
{workspace_root}/agents/{layer}/...             ← priority 2 (workspace mode only)
cortex/agents/{layer}/...                       ← priority 3 (default)
```

**Per-layer merge semantics:**

| Layer | Semantic |
|---|---|
| `workflows/` | Replacement (preserved from v0.1) |
| `roles/` | Additive |
| `capabilities/` | Additive |
| `personalities/{theme}/theme.md` | Additive |
| `personalities/{theme}/{character}.md` | Additive |
| `personalities/{theme}/characters.md` | **Not overridable** (fork the theme instead) |

### 📜 Overlay file convention

Every overlay starts with a header that names its base, scope, and merge semantic:

```markdown
<!-- OVERLAY
     Base: cortex/agents/roles/engineering/lead-backend.md
     Scope: service @backend
     Semantic: additive
-->
```

Sections inside the overlay are tagged `(additive)` so the structure is unambiguous.

### ✅ `validate-overlays.sh`

A new script (`cortex/bin/validate-overlays.sh`) checks overlay integrity:

- Header presence and required fields
- `Base:` points to an existing file
- Overlay path mirrors base path
- `Semantic:` is valid for the layer
- Non-overridable files are not overridden

CI-friendly: exit code 0/1.

- Updated: bootstrap templates teach the cascade and now wrap the personality block in `<!-- PERSONALITY:BEGIN/END -->` markers for clean stripping when `--no-personality` is used
- Refactored: [`setup.sh`](setup.sh) — no longer hardcodes the bootstrap content; reads it from the appropriate template and respects `--no-personality`. Fixes the pre-existing bug where `--workspace` mode wrote a content that didn't match the workspace template.
- **New: dynamic theme resolution via `.active-theme` marker** — the bootstrap templates no longer hardcode `h2g2`. The PM reads `cortex/agents/personalities/.active-theme` at boot to discover the active theme, then walks the cascade (cortex → workspace overlay → service overlay) to load the theme files. The marker lives **inside cortex** (it's cortex's own state) and is excluded by cortex's `.gitignore` — each developer's choice stays local without polluting git. Switching themes is now `echo my-theme > cortex/agents/personalities/.active-theme`, no setup re-run. Setting the marker to `none` disables the personality layer dynamically.
- **New: fully custom themes (and other custom additions) supported** — a host project can drop a brand-new theme into its overlay tree (`agents/personalities/my-theme/`) without any cortex base, set the name in the marker, and the PM picks it up. The same pattern works for custom roles and capabilities. Files without an `<!-- OVERLAY -->` header are treated as **custom additions** (not overlays); `validate-overlays.sh` logs them as informational and skips overlay-specific checks.
- Documentation: dual installation modes (Git submodule for single project / monorepo, standalone clone for multi-repo workspace) now first-class throughout README, getting-started, CONTRIBUTING, ADR-001.

## Breaking changes

None for end users. Overlay support is **additive**: projects without overlay folders behave exactly as in v0.1.

## Migration notes

If you have an existing v0.1 project with project-specific workflows in `agents/workflows/`: **nothing to do**. The cascade is backward compatible.

If you want to add role/capability/personality overlays:

1. Read [docs/extending-layers.md](docs/extending-layers.md)
2. Create your overlay file at the mirrored path
3. Add the overlay header
4. Run `./cortex/bin/validate-overlays.sh` to confirm
