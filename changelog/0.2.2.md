# 🚀 Cortex v0.2.2 — Cortex Runtime direction (ADR-002)

> *"I'd far rather be happy than right any day."* — Slartibartfast. A documentation patch that decides where Cortex goes next.

## Highlights

### 🏗️ ADR-002 — Cortex Runtime (Proposed)

Adds **[ADR-002 — Cortex Runtime: deployable engine + runtime project binding](docs/adr/ADR-002-cortex-runtime.md)** (status *Proposed*). It sets the direction for turning Cortex from a design-time spec read by an LLM into a **deployable engine** that executes it — **no code ships in this patch**, it is the contract the future implementation will follow.

Key points:
- **Firewall** — the runtime *consumes* the spec; the spec never depends on the runtime (the Markdown cascade stays portable).
- **ADR-001 compiled** — `resolveLayer()` becomes an executable resolver (shared with `validate-overlays.sh`).
- **Agnostic API** — generic `POST /run` + project-declared domain endpoints.
- **Runtime project binding** — CI-runner model: named warm mirrors (multi-tenant), `git worktree` for concurrency, read mode vs build mode (reuse host CI via branch push).
- **Build vs Buy** — build the resolver + thin API; embed the Agent SDK and an optional LiteLLM gateway; reuse existing CI.
- **Secrets** — swappable backend (K8s Secrets → HashiCorp Vault), per-tenant, least privilege.
